# stackoverflow-parser

Скачивает вопросы с Stack Overflow по заданным тегам через [Search API](https://api.stackexchange.com/docs/search) и сохраняет результаты в JSON файлы.

## Требования

- JDK 17+
- sbt 1.9+

## Быстрый старт

```bash
# 1. Укажи теги — по одному на строку
echo "scala\nzio" > tags.txt

# 2. Запусти
make run
```

Результаты появятся в `./output/<tag>/page_N.json`.

## Конфигурация

Все параметры задаются через переменные окружения. Дефолты прописаны в `.env`.

| Переменная | По умолчанию | Описание |
|---|---|---|
| `SO_TAGS_FILE` | `tags.txt` | Путь к файлу с тегами |
| `SO_OUTPUT_PATH` | `./output` | Куда сохранять результаты |
| `SO_PAGES` | `5` | Сколько страниц скачать на каждый тег |
| `SO_PAGE_SIZE` | `10` | Размер страницы (макс. 100) |
| `SO_SORT` | `activity` | Сортировка: `activity`, `votes`, `creation`, `relevance` |
| `SO_ORDER` | `desc` | Порядок: `desc`, `asc` |
| `SO_MAX_PARALLEL` | `3` | Максимум параллельных запросов |

## Лимиты API

Без API ключа квота составляет **300 запросов в сутки** на IP. Это мало — 5 тегов по 5 страниц уже 25 запросов, прочел это на основной странице.

API также может вернуть поле `backoff` в ответе — это сигнал подождать N секунд перед следующим запросом. Парсер обрабатывает это автоматически.

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| 404 | Страница пропускается, выводится предупреждение |
| 403 | Программа останавливается (плохой ключ или квота исчерпана) |
| 500 | Программа останавливается |

## Docker

```bash
sbt assembly
docker build -t stackoverflow-parser .
docker run --rm \
  --env-file .env \
  -v "$(pwd)/tags.txt:/app/tags.txt" \
  -v "$(pwd)/output:/data/results" \
  stackoverflow-parser
```
