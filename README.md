# stackoverflow-parser

Скачивает вопросы с Stack Overflow по заданным тегам через [Search API](https://api.stackexchange.com/docs/search) и сохраняет результаты в JSON файлы.

## Требования

- JDK 17+
- sbt 1.9+

## Быстрый старт

```bash
# 1. Скопируй конфиг
cp .env.example .env

# 2. Укажи теги — по одному на строку
echo "scala\nzio" > tags.txt

# 3. Запусти
make run
```

Результаты появятся в `./output/<tag>/page_N.json`.

## Конфигурация

Все параметры задаются через переменные окружения в файле `.env`.

| Переменная | По умолчанию | Описание |
|---|---|---|
| `SO_TAGS_FILE` | `tags.txt` | Путь к файлу с тегами |
| `SO_OUTPUT_PATH` | `./output` | Куда сохранять результаты |
| `SO_PAGES` | `2` | Сколько страниц скачать на каждый тег |
| `SO_PAGE_SIZE` | `10` | Размер страницы (макс. 100) |
| `SO_SORT` | `activity` | Сортировка: `activity`, `votes`, `creation`, `relevance` |
| `SO_ORDER` | `desc` | Порядок: `desc`, `asc` |
| `SO_MAX_PARALLEL` | `3` | Максимум одновременных HTTP-запросов суммарно |

`SO_MAX_PARALLEL` ограничивает общее число параллельных запросов — и по тегам, и по страницам внутри тега.

## Лимиты API

Без API ключа квота составляет **300 запросов в сутки** на IP.

Считай так: `SO_PAGES × кол-во тегов = запросов на запуск`. При дефолтных значениях и 2 тегах — 4 запроса за запуск, квоты хватит надолго. Если поднять `SO_PAGES=10` и добавить 10 тегов — уже 100 запросов за раз, квота кончится за 3 запуска.

API также может вернуть поле `backoff` в ответе — сигнал подождать N секунд. Парсер обрабатывает это автоматически.

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| 404 | Страница пропускается, выводится `[SKIP]` |
| 403 | Программа останавливается, выводится `[ERROR]` |
| 500 | Программа останавливается |

## Docker

```bash
sbt assembly
docker build -t stackoverflow-parser .
docker run --rm \
  --env-file .env \
  -v "$(pwd)/tags.txt:/app/tags.txt" \
  -v "$(pwd)/output:/data/results" \
  stackoverflow-parser
```
